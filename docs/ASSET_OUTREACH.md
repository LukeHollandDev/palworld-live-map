# Map asset outreach

The current map files are transformed exports of native 8192×8192 textures in a locally installed Palworld 1.0 client. They do not use TH.GL images or tiles. Pocketpair still owns the underlying artwork, and local extraction alone does not establish redistribution permission.

## Pocketpair licensing enquiry

Submit through <https://palworld.co.jp/en/contact/form/>:

> I maintain Palworld Live Map, an open-source, read-only companion for self-hosted dedicated servers. It displays live player positions from the official REST API and requires no client mod. The container currently includes JPEG conversions of the in-game `T_WorldMap` and `T_TreeMap` textures so operators do not need a separate game installation. The project is unofficial, non-commercial, attributes Pocketpair, and has a prompt removal process. May we redistribute these transformed map backgrounds in the public GitHub repository and container images? If so, please tell us the required attribution, branding, and usage conditions. If not, we will distribute only a tool that lets each operator export them from their own installed copy.

Include the repository URL and do not claim that extraction, attribution, or another fan site's usage already grants permission.

## TH.GL source enquiry

Send to the public contact address in TH.GL's legal notice:

> I maintain an open-source live map for self-hosted Palworld dedicated servers. Earlier development builds assembled background tiles from TH.GL, but those files have now been completely replaced by exports from my own installed Palworld client. I am documenting provenance and would appreciate any public guidance you can share: are TH.GL's Palworld map backgrounds direct game exports, original redraws, or used under permission from Pocketpair? I will not represent a lack of reply as permission, and I am not asking to reuse your files.

Record the date, exact message, and response privately. If Pocketpair denies redistribution or submits a valid takedown request, remove the bundled images and switch to the documented user-side export fallback.
